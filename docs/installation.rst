############
Installation
############

``skycam_utils`` is a standard PEP 621 / ``pyproject.toml`` package. It is not
published on PyPI, and the version is generated by ``setuptools_scm`` at
build/install time, so install from a git checkout rather than a source tarball
without ``.git``.

First clone the repository from GitHub
(`tepickering/skycam-utils <https://github.com/tepickering/skycam-utils>`_):

.. code-block:: bash

   git clone https://github.com/tepickering/skycam-utils.git
   cd skycam-utils

Then install from the checkout:

.. code-block:: bash

   # Editable install with the test extras
   pip install -e ".[test]"

   # Just the runtime package
   pip install .

Optional dependency groups
===========================

- ``test`` — ``pytest`` and the astropy test plugins; run the suite with
  ``pytest``.
- ``docs`` — ``sphinx-astropy`` (which pulls in ``sphinx-automodapi``,
  ``numpydoc``, and the astropy Sphinx theme). Build the docs with:

  .. code-block:: bash

     pip install ".[docs]"
     sphinx-build -b html docs docs/_build/html

- ``extra`` — Jupyter / IPython tooling for interactive exploration.

External tools
==============

Two capabilities shell out to binaries that are **not** Python dependencies and
must be installed separately:

- **astrometry.net** — :func:`skycam_utils.astrometry.solve_field` invokes
  ``solve-field`` (used by the ASI pipeline on a central image cutout). It
  requires the ``solve-field`` binary on ``PATH`` plus appropriate index files.
  This is only used by the ASI pipeline, so it is not a hard dependency of the package.
- **redis** — only needed by the operational publishers under ``scripts/`` (TCS
  state and weather telemetry), not by the photometry pipeline. This is used in
  operational scripts that run on the MMT's internal network, so it is not a hard
  dependency of the package.

Calibration assets
==================

Per-epoch and per-date calibration data ship inside the package
(``skycam_utils/data/``): the Stellacam WCS/mask FITS, the Alcor bright-star
catalogs, the bad-pixel masks (``data/badpix/``), and the horizon masks
(``data/horizon/``). The Alcor *geometry* and *photometric* constants live in
the time-indexed ``ALCOR_CALIBRATIONS`` and ``ALCOR_ZEROPOINTS`` tables in
:mod:`skycam_utils.alcor` and are resolved automatically by frame epoch.
